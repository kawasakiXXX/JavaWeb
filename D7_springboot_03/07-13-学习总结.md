### 一、Spring Boot 文件上传

#### 1.1 核心类：`MultipartFile`

`MultipartFile` 是 Spring 提供的用于处理文件上传的接口，封装了上传文件的所有信息。

**常用方法：**

| 方法 | 说明 |
|------|------|
| `getOriginalFilename()` | 获取原始文件名 |
| `transferTo(File dest)` | 将上传的文件保存到目标路径 |
| `getSize()` | 获取文件大小（字节） |
| `getContentType()` | 获取文件 MIME 类型 |
| `isEmpty()` | 判断文件是否为空 |

#### 1.2 文件上传 Controller 示例

```java
@PostMapping("/upload")
public Result upload(String name, Integer age, MultipartFile file) throws Exception {
    // 1. 获取原始文件名
    String originalFilename = file.getOriginalFilename();

    // 2. 构造唯一文件名（防止重名覆盖）
    String suffix = originalFilename.substring(originalFilename.lastIndexOf("."));
    String fileName = UUID.randomUUID().toString() + suffix;

    // 3. 保存文件到磁盘
    file.transferTo(new File("D:\\SpringBoot文件上传\\" + fileName));

    return Result.success();
}
```

#### 1.3 前端 HTML 表单

```html
<form action="/upload" method="post" enctype="multipart/form-data">
    姓名 : <input type="text" name="name"><BR>
    年龄 : <input type="text" name="age"><BR>
    头像 : <input type="file" name="file"><BR>
    <input type="submit" value="上传">
</form>
```

**关键点：**
- `method="post"` — 文件上传必须用 POST
- `enctype="multipart/form-data"` — 文件上传必须设置此编码类型
- `<input type="file">` — 文件选择控件

#### 1.4 文件上传配置（application.yml）

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 10MB       # 单个文件最大 10MB
      max-request-size: 100MB   # 整个请求最大 100MB（含所有文件和表单数据）
```

> **如果不配置，Spring Boot 默认单个文件最大 1MB，超出会抛出 `MaxUploadSizeExceededException`。**

#### 1.5 UUID 生成唯一文件名

```java
String fileName = UUID.randomUUID().toString() + suffix;
```

- `UUID.randomUUID()` 生成全局唯一标识符（如 `a1b2c3d4-e5f6-7890-abcd-ef1234567890`）
- 防止用户上传同名文件导致相互覆盖

### 二、阿里云 OSS 对象存储

#### 2.1 什么是 OSS？

OSS（Object Storage Service）是阿里云提供的海量、安全、低成本的云存储服务，适合存储图片、视频、文档等静态文件。

#### 2.2 Maven 依赖

```xml
<!-- 阿里云 OSS SDK -->
<dependency>
    <groupId>com.aliyun.oss</groupId>
    <artifactId>aliyun-sdk-oss</artifactId>
    <version>3.18.4</version>
</dependency>

<!-- OSS SDK 需要的 JAXB 相关依赖（Java 9+ 需要） -->
<dependency>
    <groupId>javax.xml.bind</groupId>
    <artifactId>jaxb-api</artifactId>
    <version>2.3.1</version>
</dependency>
<dependency>
    <groupId>javax.activation</groupId>
    <artifactId>activation</artifactId>
    <version>1.1.1</version>
</dependency>
<dependency>
    <groupId>org.glassfish.jaxb</groupId>
    <artifactId>jaxb-runtime</artifactId>
    <version>2.3.3</version>
</dependency>
```

> **注意：** Java 9+ 移除了 `javax.xml.bind` 模块，使用 OSS SDK 需要手动引入 JAXB 依赖。

#### 2.3 核心 API 使用流程

```java
// 1. 配置 Endpoint（访问域名）
String endpoint = "https://oss-cn-beijing.aliyuncs.com";

// 2. 配置访问凭证（推荐使用环境变量方式，安全性更高）
EnvironmentVariableCredentialsProvider credentialsProvider =
    CredentialsProviderFactory.newEnvironmentVariableCredentialsProvider();

// 3. 创建 OSSClient 实例
ClientBuilderConfiguration config = new ClientBuilderConfiguration();
config.setSignatureVersion(SignVersion.V4);  // 使用 V4 签名
OSS ossClient = OSSClientBuilder.create()
    .endpoint(endpoint)
    .credentialsProvider(credentialsProvider)
    .clientConfiguration(config)
    .region("cn-beijing")
    .build();

// 4. 上传文件（3种方式）
// 方式一：上传文件流
byte[] content = Files.readAllBytes(file.toPath());
ossClient.putObject(bucketName, objectName, new ByteArrayInputStream(content));

// 方式二：上传本地文件路径
// ossClient.putObject(bucketName, objectName, new File("path/to/file"));

// 方式三：上传字符串
// ossClient.putObject(bucketName, objectName, new ByteArrayInputStream("content".getBytes()));

// 5. 关闭客户端（释放资源）
ossClient.shutdown();
```

#### 2.4 OSS 核心概念

| 概念 | 说明 | 类比 |
|------|------|------|
| **Bucket** | 存储空间，存放对象的容器 | 相当于"文件夹" |
| **Endpoint** | 访问域名，格式：`https://oss-cn-{region}.aliyuncs.com` | 相当于"网址" |
| **Object** | 存储的对象（文件） | 相当于"文件" |
| **Region** | 数据中心所在地域，如 `cn-beijing` | 相当于"机房位置" |
| **AccessKey** | 访问密钥，用于身份认证 | 相当于"账号密码" |

#### 2.5 OSS Client 创建方式（新版 V4 签名）

```java
// 使用 Builder 模式创建（推荐，支持 V4 签名）
OSS ossClient = OSSClientBuilder.create()
    .endpoint(endpoint)
    .credentialsProvider(credentialsProvider)
    .clientConfiguration(clientBuilderConfiguration)
    .region(region)
    .build();
```

> OSS SDK 共有两种创建 Client 的方法：
> - **老方法：** `new OSSClientBuilder().build(endpoint, ak, sk)` — 简单但不够灵活
> - **新方法：** `OSSClientBuilder.create()...build()` — Builder 模式，支持 V4 签名、环境变量凭证等

#### 2.6 环境变量方式管理凭证（推荐）

```java
EnvironmentVariableCredentialsProvider credentialsProvider =
    CredentialsProviderFactory.newEnvironmentVariableCredentialsProvider();
```

需要在系统中设置环境变量：
- `OSS_ACCESS_KEY_ID` — AccessKey ID
- `OSS_ACCESS_KEY_SECRET` — AccessKey Secret

> **安全性：** 使用环境变量管理密钥，避免将敏感信息硬编码在代码中或提交到 Git 仓库。

## 📚 项目整体知识体系回顾

### D1 — Spring Boot 入门与 HTTP 基础

| 知识点 | 说明 |
|--------|------|
| `@RestController` | 标识当前类为请求处理类（= `@Controller` + `@ResponseBody`） |
| `@RequestMapping` | 映射 HTTP 请求路径到方法 |
| `HttpServletRequest` | 获取请求方式、URL、请求头、请求参数 |
| `HttpServletResponse` | 设置响应状态码、响应头、响应体 |
| `ResponseEntity` | Spring 提供的响应封装类，链式调用设置完整响应 |

### D2/D3 — 三层架构与 IoC/DI

| 知识点 | 说明 |
|--------|------|
| **三层架构** | Controller（控制层） → Service（业务层） → DAO（数据层） |
| **IoC（控制反转）** | 对象的创建和管理交给 Spring 容器，从"new 对象"变为"容器注入" |
| **DI（依赖注入）** | 容器将 Bean 注入到需要它的地方 |
| **`@Component` 系列** | `@Controller` / `@Service` / `@Repository` — 将类注册为 Bean |
| **`@Configuration` + `@Bean`** | 显式声明 Bean（适合第三方类），`@Configuration` 有 CGLIB 代理保证单例 |
| **`@Scope`** | Bean 作用域：`singleton`（默认单例） / `prototype`（每次新的） |
| **Bean 生命周期** | 构造器 → `@PostConstruct` 初始化 → 运行中 → `@PreDestroy` 销毁 |
| **`@Autowired`** | 自动装配 Bean |

**三种注入方式对比：**

| 方式 | 写法 | 推荐度 |
|------|------|--------|
| **构造器注入** | `private final Xxx xxx;` + 构造器 | ✅ 推荐（不可变，启动即报错） |
| **Setter 注入** | `@Autowired` + setter 方法 | 可选依赖场景 |
| **字段注入** | `@Autowired private Xxx xxx;` | ⚠️ 不推荐（不能 final，难测试） |

### D4 — JDBC 基础

```java
// JDBC 标准5步操作
Class.forName("com.mysql.cj.jdbc.Driver");  // 1. 加载驱动
Connection conn = DriverManager.getConnection(url, user, password); // 2. 获取连接
Statement stmt = conn.createStatement();     // 3. 获取操作对象
int n = stmt.executeUpdate(sql);             // 4. 执行 SQL
stmt.close(); conn.close();                  // 5. 释放资源
```

### D6 — Spring Boot CRUD 完整项目

| 知识点 | 说明 |
|--------|------|
| **MyBatis** | 半自动 ORM 框架，SQL 由开发者手写 |
| **`@Mapper`** | 标记接口为 MyBatis Mapper，运行时自动生成代理对象并注册到 IoC |
| **`@Select/@Insert/@Update/@Delete`** | MyBatis 注解式 SQL |
| **`#{param}`** | MyBatis SQL 占位符，防 SQL 注入 |
| **驼峰映射** | `map-underscore-to-camel-case: true`，自动将 `create_time` → `createTime` |
| **RESTful API** | `GET` 查询 / `POST` 新增 / `PUT` 更新 / `DELETE` 删除 |
| **`@PathVariable`** | 从 URL 路径中获取参数（如 `/users/{id}`） |
| **`@RequestBody`** | 将请求体 JSON 转为 Java 对象 |
| **Lombok** | `@Data` / `@AllArgsConstructor` / `@NoArgsConstructor` 简化 JavaBean 编写 |
| **统一响应结果** | 封装 `Result` 类（code + msg + data），统一接口返回格式 |
| **`@Slf4j`** | Lombok 日志注解，自动生成 `log` 对象 |
| **三层架构实践** | Controller → Service（接口+实现） → Mapper → MySQL |

## 🔑 今日关键总结

1. **文件上传三要素：** POST 请求 + `multipart/form-data` + `MultipartFile` 接收
2. **防文件名冲突：** 用 `UUID.randomUUID()` 生成唯一文件名
3. **OSS 上传流程：** 创建凭证 → 构建 Client → putObject → shutdown
4. **安全管理密钥：** 用环境变量管理 AccessKey，不要硬编码
5. **Java 9+ 兼容：** 使用 OSS SDK 需要额外引入 JAXB 依赖
6. **Builder 模式：** 新版 OSS Client 使用 `OSSClientBuilder.create()...build()` 链式构建
