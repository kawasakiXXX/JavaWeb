## 一、Spring Boot 配置优先级

Spring Boot 的配置来源众多，加载优先级由高到低大致如下：

| 优先级 | 配置来源 | 说明 |
|--------|----------|------|
| 1 | 命令行参数 `--server.port=8080` | 启动时传入，最高优先级 |
| 2 | `spring.application.json` | JNDI / Java 系统属性中的 JSON 配置 |
| 3 | Servlet 初始化参数 | `ServletConfig` 中的参数 |
| 4 | JNDI 属性 | `java:comp/env` |
| 5 | Java 系统属性 `System.getProperties()` | `-Dkey=value` |
| 6 | OS 环境变量 | 如 `SERVER_PORT` |
| 7 | `application-{profile}.properties/yml` | 特定环境的配置文件 |
| 8 | `application.properties/yml` (jar 外部) | jar 包同目录下的配置文件 |
| 9 | `application.properties/yml` (jar 内部) | classpath 下的配置文件 |
| 10 | `@PropertySource` 注解 | 自定义属性文件 |
| 11 | 默认属性 | `SpringApplication.setDefaultProperties()` |

### YAML vs Properties
- **properties**: `key=value` 形式，优先级略高于 YAML
- **yaml/yml**: 层级结构，可读性更好，支持复杂数据结构（数组、对象）

## 二、Bean 作用域 (Scope)

Spring 容器管理 Bean 的作用域，通过 `@Scope` 注解指定：

| 作用域 | 说明 | 使用场景 |
|--------|------|----------|
| **singleton** (默认) | 整个容器中只有一个实例 | 无状态 Service、DAO、配置类 |
| **prototype** | 每次获取都创建新实例 | 有状态的 Bean，如控制器中携带请求数据的对象 |
| **request** | 每个 HTTP 请求一个实例 | Web 环境下请求级数据 |
| **session** | 每个 HTTP Session 一个实例 | 用户会话数据 |
| **application** | 每个 ServletContext 一个实例 | 全局共享的 Web 数据 |

### 示例代码

```java
@Component
@Scope("prototype")  // 每次注入都创建新实例
public class OrderService {
    // ...
}

@Component
@Scope(ConfigurableBeanFactory.SCOPE_SINGLETON)  // 默认，可省略
public class UserService {
    // ...
}
```

### 注意事项
- singleton 中注入 prototype 时，prototype 只会创建一次（需要特殊处理如 `ObjectFactory`、`@Lookup` 或代理模式）
- Web 作用域需要对应的 Web 环境支持

## 三、Bean 生命周期

Spring Bean 从创建到销毁经历以下阶段：

```
实例化 → 属性注入 → Aware 回调 → 初始化前处理 → 初始化 → 初始化后处理 → 就绪 → 销毁
```

### 3.1 详细阶段

| 阶段 | 回调方式 | 说明 |
|------|----------|------|
| **实例化** | 构造方法 | 通过反射创建对象实例 |
| **属性注入** | `@Autowired` / Setter | 依赖注入 |
| **Aware 系列** | `BeanNameAware` / `BeanFactoryAware` / `ApplicationContextAware` | 让 Bean 感知容器信息 |
| **前置处理** | `BeanPostProcessor.postProcessBeforeInitialization()` | 在初始化之前执行 |
| **初始化** | `@PostConstruct` / `InitializingBean.afterPropertiesSet()` / `@Bean(initMethod="...")` | 自定义初始化逻辑 |
| **后置处理** | `BeanPostProcessor.postProcessAfterInitialization()` | 代理增强（如 AOP） |
| **销毁** | `@PreDestroy` / `DisposableBean.destroy()` / `@Bean(destroyMethod="...")` | 资源释放 |

### 3.2 代码示例

```java
@Component
public class LifecycleBean implements InitializingBean, DisposableBean, BeanNameAware {

    @PostConstruct
    public void postConstruct() {
        System.out.println("1. @PostConstruct 初始化");
    }

    @Override
    public void afterPropertiesSet() throws Exception {
        System.out.println("2. InitializingBean.afterPropertiesSet()");
    }

    @Bean(initMethod = "customInit")
    public void customInit() {
        System.out.println("3. @Bean initMethod");
    }

    @PreDestroy
    public void preDestroy() {
        System.out.println("4. @PreDestroy 销毁前");
    }

    @Override
    public void destroy() throws Exception {
        System.out.println("5. DisposableBean.destroy()");
    }
}
```

### 3.3 执行顺序
1. `@PostConstruct`
2. `InitializingBean.afterPropertiesSet()`
3. `@Bean initMethod`
4. `@PreDestroy`
5. `DisposableBean.destroy()`
6. `@Bean destroyMethod`

## 四、第三方 Bean 的管理

### 4.1 @Bean 手动注册

最直接的方式，在 `@Configuration` 类中通过 `@Bean` 方法注册第三方对象：

```java
@Configuration
public class DataSourceConfig {
    @Bean
    public DataSource dataSource() {
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:mysql://localhost:3306/mydb");
        ds.setUsername("root");
        ds.setPassword("123456");
        return ds;
    }
}
```

### 4.2 @Import 导入类

将指定的类直接注册为 Bean，常用于导入配置类或普通组件：

```java
@Import({MyConfig.class, MyComponent.class})
@SpringBootApplication
public class Application { }

// 配合 ImportSelector 实现条件化批量导入
public class MyImportSelector implements ImportSelector {
    @Override
    public String[] selectImports(AnnotationMetadata metadata) {
        return new String[]{"com.example.MyBean1", "com.example.MyBean2"};
    }
}
```

### 4.3 @ComponentScan 包扫描

指定扫描路径，将路径下的 `@Component` / `@Service` / `@Repository` / `@Controller` 注册为 Bean：

```java
@ComponentScan(basePackages = {"com.example.service", "com.third.party.lib"})
@SpringBootApplication
public class Application { }
```

### 4.4 @EnableXxx 开关注解

一种设计模式：`@EnableXxx` 通常组合了 `@Import`，用于一键开启某项功能：

```java
@EnableScheduling    // 开启定时任务
@EnableAsync         // 开启异步处理
@EnableTransactionManagement  // 开启事务管理
@EnableCaching       // 开启缓存
public class Application { }
```

设计模式：
```
@EnableXxx → @Import({XxxConfigurationSelector.class})
           → 选择性导入 @Configuration 配置类
           → 配置类中 @Bean 注册所需组件
```

### 4.5 @ConditionalOnXxx 条件装配

根据条件决定是否注册 Bean，是 Spring Boot 自动配置的核心机制：

| 注解 | 条件说明 |
|------|----------|
| `@ConditionalOnClass` | 类路径中存在指定类时才生效 |
| `@ConditionalOnMissingClass` | 类路径中不存在指定类时才生效 |
| `@ConditionalOnBean` | 容器中存在指定 Bean 时才生效 |
| `@ConditionalOnMissingBean` | 容器中不存在指定 Bean 时才生效 |
| `@ConditionalOnProperty` | 指定配置属性存在且有值时生效 |
| `@ConditionalOnResource` | 存在指定资源文件时生效 |
| `@ConditionalOnWebApplication` | 当前是 Web 应用时生效 |
| `@ConditionalOnExpression` | SpEL 表达式为 true 时生效 |
| `@ConditionalOnJava` | JDK 版本满足条件时生效 |

### 示例

```java
@Configuration
@ConditionalOnClass({DataSource.class})      // 有数据源类时生效
@ConditionalOnProperty(prefix = "my.db", name = "enabled", havingValue = "true", matchIfMissing = false)
public class DatabaseAutoConfiguration {
    @Bean
    @ConditionalOnMissingBean               // 用户没自己定义时才注册
    public DataSource dataSource() {
        return new HikariDataSource();
    }
}
```

## 五、自定义 Starter

自定义 Starter 实现两大核心功能：**依赖管理** + **自动配置**。

### 5.1 Starter 命名规范

- 官方：`spring-boot-starter-xxx`
- 第三方 / 自定义：`xxx-spring-boot-starter`

### 5.2 模块结构

```
my-starter/
├── my-spring-boot-starter/          # 空的 starter 模块（只做依赖聚合）
│   └── build.gradle / pom.xml       # 引入 autoconfigure 模块
│
└── my-spring-boot-starter-autoconfigure/  # 自动配置模块
    ├── src/main/java/
    │   └── com/example/
    │       ├── MyAutoConfiguration.java    # 自动配置类
    │       └── MyProperties.java           # 配置属性类
    │
    └── src/main/resources/
        └── META-INF/
            └── spring/
                └── org.springframework.boot.autoconfigure.AutoConfiguration.imports  (Spring Boot 3.x)
                # 或 spring.factories (Spring Boot 2.x)
```

### 5.3 依赖管理 (starter 模块)

`pom.xml` (只做依赖聚合，不写代码)：

```xml
<dependencies>
    <dependency>
        <groupId>com.example</groupId>
        <artifactId>my-spring-boot-starter-autoconfigure</artifactId>
        <version>1.0.0</version>
    </dependency>
</dependencies>
```

### 5.4 自动配置 (autoconfigure 模块)

#### ① 配置属性类

```java
@ConfigurationProperties(prefix = "my.starter")
@Data
public class MyProperties {
    private boolean enabled = true;
    private String name = "default";
    private int timeout = 5000;
}
```

#### ② 核心业务类

```java
public class MyService {
    private MyProperties properties;

    public MyService(MyProperties properties) {
        this.properties = properties;
    }

    public String sayHello() {
        return "Hello, " + properties.getName();
    }
}
```

#### ③ 自动配置类

```java
@Configuration
@EnableConfigurationProperties(MyProperties.class)
@ConditionalOnProperty(prefix = "my.starter", name = "enabled", havingValue = "true", matchIfMissing = true)
public class MyAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public MyService myService(MyProperties properties) {
        return new MyService(properties);
    }
}
```

#### ④ 自动配置入口声明

`META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`：
```
com.example.MyAutoConfiguration
```

> **Spring Boot 2.x** 使用 `META-INF/spring.factories`：
> ```properties
> org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
> com.example.MyAutoConfiguration
> ```

### 5.5 使用自定义 Starter

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>my-spring-boot-starter</artifactId>
    <version>1.0.0</version>
</dependency>
```

```properties
# application.properties
my.starter.name=Spring Boot
my.starter.timeout=10000
```

## 六、知识脉络总结

```
Spring Boot 核心能力
├── 配置体系
│   ├── 配置优先级（命令行 > 环境变量 > 配置文件 > 默认值）
│   └── 配置格式（properties / yaml）
│
├── Bean 管理
│   ├── 作用域（singleton / prototype / request / session）
│   └── 生命周期（实例化 → 注入 → 初始化 → 就绪 → 销毁）
│
├── 第三方集成
│   ├── 注册方式（@Bean / @Import / @ComponentScan）
│   ├── 开关注解（@EnableXxx 设计模式）
│   └── 条件装配（@ConditionalOnXxx）
│
└── 自定义 Starter
    ├── 依赖管理（starter 模块做依赖聚合）
    └── 自动配置（autoconfigure 模块 + @Conditional + META-INF 声明）
```
